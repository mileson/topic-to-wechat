# 大纲护栏规则

## 目的

大纲是防止跑题的核心护栏。生成大纲后自动进入下一阶段，无需用户确认。

## 大纲结构

```yaml
标题: [Stage 2 自动选定的标题]
选题: [选题关键词]
目标读者: [推断的读者画像]
核心论点: [一句话概括本文要传递的核心信息]

sections:
  - heading: "H2 标题"
    key_points:
      - 要点 1
      - 要点 2
    data_sources:
      - 来源描述（如 "WebSearch 获取的 X 公司公告"）
    estimated_words: 200-400

  - heading: "..."
    ...
```

## 硬性规则

1. **分区数量**：3-5 个 H2 分区（不含开头和结尾）
2. **每区要素**：必须包含 key_points + data_sources
3. **开头段**：1-2 句话概括全文要点，不单独成 H2
4. **结尾段**：可选，1 段总结 + 展望，不单独成 H2
5. **总字数**：大纲阶段估算正文总字数在 1500-3000 字之间

## 防虚构检查（条件触发）

根据题材类型决定检查严格程度：

### 强制检查（新闻/数据/事实陈述类）

- 涉及具体数据、公司公告、产品发布等事实性内容时
- 每条论断必须有搜索结果支撑
- 无法验证的信息标注 `[待验证]`，正文写作时必须通过 WebSearch 核实

### 可选检查（故事/观点/经验分享类）

- 以个人经历、踩坑复盘、观点输出为主的内容
- 事实性陈述仍需有依据，但允许合理推测和经验总结
- 不要求每个 key_point 都有严格的外部来源

### 自检判断逻辑

```
题材判断 → 新闻/数据类 → 强制检查
         → 故事/观点类 → 可选检查
         → 混合类 → 事实部分强制检查，观点部分可选
```

生成大纲时自检：
- [ ] 每个事实性 key_point 是否有对应的 data_source？
- [ ] 是否存在凭空编造的人名、公司名、产品名？
- [ ] 引用数据是否标注了来源？
