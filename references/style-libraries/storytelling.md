# 故事描述

## 定位

从经历、过程、场景和转折切入，用故事带出观点和价值。适合沉浸式阅读场景，读者期望获得代入感和启发。

## 适用内容

- 个人经历（使用某个工具的前后对比、转变过程）
- 开发过程（从 0 到 1 搭建的完整记录）
- 踩坑复盘（遇到问题 → 尝试 → 解决 → 收获）
- 有明显时间线或转折的主题

## 开头方式

- 从一个具体场景开场（"那天晚上 11 点，第三次部署失败的时候…"）
- 从一次失败、卡住、误判开始
- 用"我当时以为…后来发现…"引入

## 结构方式

1. 先给场景或冲突（制造共鸣）
2. 再说明为什么会卡住（建立认同）
3. 再带出转折和解决方式（核心价值）
4. 最后收束成观点或方法（升华）

## 语气要求

- 有画面感，读者能"看到"场景
- 有过程感，不是直接给答案
- 真诚，像朋友在聊天
- 情绪服务于信息，不喧宾夺主

## 禁止写法

- 铺垫太长，迟迟不进入主题
- 只有感受，没有信息增量
- 把故事写成流水账（"然后…然后…然后…"）
- 为了煽情而煽情

## 候选稿输出要求

- 候选稿长度控制在目标正文的 40%-60%
- 必须让读者明显感受到"过程"和"转折"
